## 一、产品概述

行业热点爬取工具，输入行业关键词，自动搜索多源热点、爬取全文、转 Markdown 存储。

一句话：**你想要监控什么关键词，它就去全网把相关内容找回来，整理成可读的文章。**

---

## 二、核心功能

### 2.1 搜索发现

输入一个关键词，同时从 9 个源搜索：

| 源 | 类型 | 说明 |
|:--:|:----|------|
| 百度热搜 | 热榜 | 百度实时热搜榜，API 直调 |
| 知乎热榜 | 热榜 | 知乎实时热榜，需扫码登录一次 |
| 必应搜索 | 网页搜索 | 搜索引擎结果 |
| 微博热搜 | 热榜 | 微博实时热搜榜 |
| 今日头条 | 热榜 | 通过 60s 免费 API |
| 360 搜索 | 网页搜索 | 搜索引擎结果 |
| 抖音热点 | 热榜 | 通过 60s 免费 API |
| B站热搜 | 热榜 | B站官方热搜 API + 热门视频 |
| 微信搜一搜 | 公众号搜索 | 搜狗微信搜索，Playwright 渲染取内容 |

### 2.2 内容提取

多种提取器投票选最优：

| 提取器 | 适用场景 |
|:------|----------|
| Readability | 标准新闻/博客文章 |
| 文本密度 | 结构松散的页面（大量 div 少 p） |
| 段落收集 | 多段落文章，按容器分组 |
| 微信专用 | 微信公众号文章，从 rich_media_content 提取 |

提取后自动清洗：去广告、去导航、去侧边栏、去跟踪参数、strip 多余属性。

### 2.3 工程保障

| 能力 | 实现 |
|:----|------|
| JS 渲染 | Playwright 智能降级（requests 失败自动走浏览器） |
| 隐身反检测 | 5 件套覆盖 webdriver/plugins/languages/chrome/permissions |
| 失败重试 | 指数退避，只在 429/5xx 重试 |
| 断点续爬 | ResumeState 追踪已完成/失败 URL |
| 内容去重 | SHA256(title+url) + SQLite 持久化 |
| 代理池 | 从 GitHub 免费代理列表拉取，自动轮换，失败剔除 |
| 登录管理 | `--login zhihu` / `--login weibo` 扫码登录，Cookie 自动缓存 |
| 多格式输出 | Markdown / JSONL / CSV |
| 结构化日志 | 三流 JSONL（request/error/summary） |

## 三、架构亮点

### 3.1 模块化设计

```
hotspot_crawler.py    ← CLI 入口
engine.py             ← 编排器（搜索→爬取→去重→存储→报告）
searcher.py           ← 多源搜索编排
sources/              ← 搜索源包（@register_source 自动注册）
fetcher.py            ← 双引擎获取器（requests + Playwright 降级）
extractor.py          ← 多提取器投票 + 清洗 DSL
proxy_manager.py      ← 动态代理池
cookie_manager.py     ← 登录态持久化
dedupe.py             ← SHA256 去重
storage.py            ← 多格式存储
run_state.py          ← 日志 + 失败恢复
```

**新增一个搜索源只需 2 步：**
1. 在 `sources/` 下新建 `.py` 文件
2. 类加 `@register_source` 装饰器

无需改 `__init__.py`，无需改配置，自动发现注册。

### 3.2 智能降级链

```
requests 直连（快）
  ↓ 内容 <200字 或 SPA 特征
Playwright 渲染（慢但全）
  ↓ Content-Type 检测
html2text 转 Markdown
  ↓ 提取器投票
Readability / 文本密度 / 段收集 / 微信专用
  ↓ 质量检查
标题有意义 + 有效字数 >= 100
  ↓ 清洗
RinseRule 去广告、去导航、去跟踪参数
```

### 3.3 失败恢复链

```
运行时异常 → 指数退避重试（up to 3 次）
  ↓ 持续失败
记录到 failed.jsonl
  ↓ 下次运行
--resume 自动加载失败列表优先重试
  ↓ 所有源独立 try/except
一个源挂掉不影响其他源
```

## 六、可拓展方向

### 6.1 加新搜索源

最简单的拓展。参考已存在的源（如 `sources/toutiao.py`）：

```python
@register_source
class MySource(SearchSource):
    name = "my_source"
    display_name = "我的源"

    def search(self, keyword, max_results=15) -> list[HotItem]:
        # 调用 API 或抓取页面
        # 返回 HotItem 列表
        ...
```

做完放 `sources/` 下即可，零配置注册。

**候选源：**
- 搜狗搜索（需解决反爬）
- 掘金（技术社区）
- CSDN（技术社区）
- 36氪（科技新闻）
- GitHub Trending（开源热点）
- 贴吧（综合话题）

### 6.2 加新提取器

在 `extractor.py` 加函数，加到 `best_extraction()` 的列表里即可。

候选：
- **Trafilatura** — 需 pip 安装，学术界广泛使用的正文提取工具
- **jina-ai/reader** — AI 驱动的页面转 Markdown
- **Mozilla Readability (Node.js)** — 通过子进程调用，效果比 python-readability 好

### 6.3 二开方向

| 方向 | 说明 | 工作量 |
|:----|------|:------:|
| **Web UI** | 加一个 Flask/FastAPI 前端，浏览器里输入关键词看结果 | 中 |
| **定时任务** | 结合 cron 定时跑关键词，攒成日报/周报 | 低 |
| **AI 摘要** | 爬完内容后调 LLM API 生成摘要和观点提炼 | 低 |
| **数据看板** | 把 JSONL 日志接入 Grafana，看爬取趋势和失败率 | 中 |
| **私域部署** | Docker 打包，定时爬取推送到飞书/钉钉/企业微信 | 中 |
| **公众号集成** | 配合 wewrite skill，爬完直接发微信公众号 | 低 |
| **动态代理升级** | 对接快代理/芝麻代理等付费服务，替代 GitHub 免费代理 | 低 |
| **视频/图片爬取** | 抖音/B站源扩展为同时爬取视频描述和封面图 | 中 |

## 八、局限性

1. **微博内容提取需登录** — 微博反爬最强，不登录拿不到正文
2. **360 搜索链接是中间页** — 真实 URL 被隐藏，需额外解析
3. **头条/抖音依赖第三方 API** — 60s API 是免费服务，可能不稳定
4. **知乎 Cookie 有时效** — 每几周需重新扫码一次
5. **免费代理质量参差不齐** — GitHub 代理池很多不可用，50 个里通常只能测出几个活的
6. **不支持翻页** — 搜索源一次性拿第一页结果
