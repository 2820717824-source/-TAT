## 三、分 Phase 执行计划

### Phase 1：JS 渲染 + 隐身（今天做）

**新增 1 个文件，修改 1 个文件：**

```
代码/
├── hotspot_crawler.py     ← 改（加 --browser 参数 + 智能降级）
├── browser_fetcher.py     ← 新增（Playwright 渲染 + 隐身）
```

**browser_fetcher.py 结构：**

```
BrowserFetcher
├── __init__(self, headless=True, stealth=True)
├── fetch(url) → html       ← 核心：隐身上下文 → goto → 返回 HTML
├── _create_context()        ← 从 image_crawler 抄隐身配置
├── scroll_and_wait()        ← 从 universal-crawler 抄滚动
└── click_expand()           ← 从 universal-crawler 抄点击"展开全文"
```

**hotspot_crawler.py 改动（4 处）：**

| # | 位置 | 改动 |
|---|------|------|
| 1 | import 区 | 加 `from browser_fetcher import BrowserFetcher`（try 包裹，可选依赖） |
| 2 | `ArticleCrawler.__init__` | 加 `self.use_browser` 参数（"auto" / "always" / "never"） |
| 3 | `crawl()` L473 之后 | requests 拿不到内容（<200 字）→ 调 BrowserFetcher 重试 |
| 4 | CLI 参数 | 加 `--browser`（`choices=['auto','always','never'], default='auto'`） |

**智能降级逻辑：**

```
crawl(url):
    if use_browser == 'never':
        → 只走 requests（老路）
    if use_browser == 'always':
        → 直接走 Playwright

    # auto 模式
    html = requests.get(url)
    if len(html) >= 200 且 不是 SPA 特征（无 __NEXT_DATA__ 等）:
        → 走老路 Readability 提取（快）
    else:
        → 走 Playwright 渲染后再提取（慢但全）
```

## 五、快速路径（如果只想解决"拿不到 JS 页面"）

如果今天时间有限，**只做 browser_fetcher.py + crawl() 加 3 行降级逻辑**：

```python
# 在 crawl() 的 requests 失败后加这段（约 15 行）
if self.use_browser != 'never':
    try:
        bf = BrowserFetcher()
        html = bf.fetch(item.url)
        # 然后走现有的 Readability 提取流程
    except Exception:
        pass  # 标记失败，不影响其他文章
```

这样改动最小，现有逻辑完全不动，只是 requests 不行时多试一次浏览器。
