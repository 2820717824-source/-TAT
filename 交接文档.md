## 一、项目概况

行业热点爬取工具。输入行业关键词，自动搜索多源热点、爬取全文、转 Markdown 存储。

### 当前状态

**全部功能已完成。** 项目从单文件 827 行 → 模块化架构（16 模块），9 个搜索源，4 个正文提取器，内置代理池、失败恢复、登录态管理。

```
├── hotspot_crawler.py    ← CLI 入口
├── engine.py             ← 编排器（搜索→爬取→去重→存储→报告）
├── searcher.py           ← 多源搜索编排（策略链模式）
├── sources/              ← 搜索源包（@register_source 自动注册）
│   ├── __init__.py       ← SearchSource 基类 + fetch_url(重试) + clean_text
│   ├── baidu.py          ← 百度热搜
│   ├── zhihu.py          ← 知乎热榜（支持 --login zhihu）
│   ├── bing.py           ← 必应搜索
│   ├── weibo.py          ← 微博热搜（支持 --login weibo）
│   ├── toutiao.py        ← 今日头条（60s API）
│   ├── douyin.py         ← 抖音热点（60s API）
│   ├── 360.py            ← 360 搜索
│   ├── bilibili.py       ← B站热搜（官方 API）
│   └── wechat.py         ← 微信搜一搜（Playwright 渲染提取）
├── fetcher.py            ← 双引擎获取器（requests + Playwright 智能降级）
├── browser_fetcher.py    ← Playwright 渲染 + 隐身配置 + Cookie 注入
├── extractor.py          ← 多提取器投票 + RinseRule 清洗 DSL
├── proxy_manager.py      ← 动态代理池（GitHub 免费代理）
├── cookie_manager.py     ← Cookie 持久化（保存/加载/验证）
├── dedupe.py             ← SHA256 去重 + SQLite
├── storage.py            ← 多格式输出（MD/JSONL/CSV）
├── run_state.py          ← TaskLogger 三流日志 + ResumeState 断点续爬
├── config.py             ← CrawlerConfig + YAML/CLI 合并
├── requirements.txt      ← 依赖
└── docs/                 ← 文档、spec、plan
```

## 二、功能详解

### 2.1 搜索发现（9 源）

| 源 | name | 类型 | 登录 |
|:--:|:----:|:----|:----:|
| 百度热搜 | baidu | 热榜 | 否 |
| 知乎热榜 | zhihu | 热榜 | `--login zhihu` 扫码 |
| 必应搜索 | bing | 网页搜索 | 否 |
| 微博热搜 | weibo | 热榜 | `--login weibo`（可选） |
| 今日头条 | toutiao | 热榜 | 否 |
| 360 搜索 | 360 | 网页搜索 | 否 |
| 抖音热点 | douyin | 热榜 | 否 |
| B站热搜 | bilibili | 热榜 | 否 |
| 微信搜一搜 | wechat | 公众号搜索 | 否（Playwright 渲染） |

新增源只需在 `sources/` 下新建文件 + `@register_source`，零配置自动注册。

### 2.2 内容提取（4 提取器投票）

| 提取器 | 适用场景 |
|:------|----------|
| Readability | 标准新闻/博客 |
| 文本密度 | 结构松散的页面 |
| 段落收集 | 多段落文章 |
| 微信专用 | 公众号文章（rich_media_content） |

加权评分：内容长度(0.35) + 段落数(0.25) + 链接密度(0.20) + 文本/HTML比(0.20)，选最优。

### 2.3 工程能力

| 能力 | 实现 |
|:----|------|
| JS 渲染 | Playwright 隐身 + SPA 检测，智能降级 |
| 失败重试 | 指数退避，仅 429/5xx 重试 |
| 断点续爬 | completed.txt + failed.jsonl，`--resume` |
| 内容去重 | SHA256(title+url) + SQLite 跨运行 |
| 代理池 | GitHub 免费代理，自动轮换，失败剔除 |
| 质量检查 | 标题验证 + 有效字数 >= 100 |
| 清洗 | RinseRule JSON DSL 去广告/导航 |
| 日志 | 三流 JSONL（request/error/summary） |
| 登录管理 | `--login zhihu` / `--login weibo` 扫码 |

### 2.4 输出

- Markdown（默认）
- JSONL（数据分析和日志）
- CSV（表格形式）
- 所有输出包含 frontmatter：title/url/source/date/keyword/status/fallback

## 四、参考项目深读报告

> 以下项目均在 `D:\js动态爬虫\` 下。

### universal-crawler ⭐ 最有价值

YAML 配置驱动的通用爬虫框架。已从中借鉴：指数退避重试、随机延迟、文本清洗、工厂注册模式、类型化配置、模块自动发现、ResumeState、TaskLogger、keep_on_detail_failed。

**关键发现：** auth 6 种类型 0 种实现——config.py 中 auth 存为裸 dict，没有任何代码读取。

### image_crawler 已充分借鉴

隐身 5 件套（browser_fetcher.py）、策略链模式（searcher.py）、双引擎降级（fetcher.py）。翻页检测 3 层策略、19 种"加载更多"点击策略尚未用。

### javaCrawling 架构启发最大

配置驱动编排（YAML 定义爬虫行为 → 动态组装）。NetPoxyService → proxy_manager.py 的灵感来源。RinseRule → extractor.py 的 rinse() 灵感来源。

### python_spider

WeiXin/ 模块的分层设计模式（config→request→db→spider→run）。Toutiao/ 的 Ajax 逆向手法。

### scrapy+phantomjs

完全过时，代码不能用。唯一概念：Downloader Middleware + 浏览器渲染的思路。

### wechat_articles_spider ✅ 新参考

微信文章提取的专精项目。核心发现：
- 标题在 `<h2>` 标签
- 正文在 `class="rich_media_content"`
- 图片用 `data-src` 而非 `src`，需替换
- URL 需含 `__biz`/`mid`/`sn`/`idx` 参数才有完整页面
- 用 `requests.get()` 即可获取，无需 Playwright（但 Sogou 反爬拦截，需 Playwright 中转）

已据此实现 `extract_wechat()`。

## 五、已解决的全部问题

| 问题 | 方案 |
|------|------|
| requests 拿不到 JS 渲染页面 | Playwright 渲染 + 隐身配置，智能降级 |
| 所有参数靠命令行传，无法固化 | YAML 配置 + CLI 合并 |
| 重复爬相同内容 | SHA256 去重 + SQLite 持久化 |
| 只有 Markdown 输出 | 支持 JSONL/CSV/MD 三种格式 |
| 流程没有统一管理 | CrawlerEngine 编排器 |
| 新增搜索源要改核心代码 | sources 包 + @register_source + importlib 自动发现 |
| 没有微博热搜源 | 新增 weibo.py |
| 网络请求失败没有重试 | 指数退避重试 |
| 文本内容有 HTML 实体和多余空白 | clean_text() 清洗工具 |
| Windows 终端 unicode 崩溃 | 替换所有 console.print 中的 unicode 符号 |
| 配置写错源名没有提示 | validate_sources() 加载时校验 |
| Cookie 类源需要手动配置 | per-source 配置透传 + CookieManager 自动管理 |
| 失败不能续爬 / 无断点续爬 | ResumeState + TaskLogger 结构化日志 |
| 无结构化运行日志 | 三流 JSONL（request/error/summary） |
| 知乎热榜需 Cookie | `--login zhihu` 扫码登录，自动缓存 |
| 内容含垃圾（[no-title]等） | 质量检查：标题验证 + 有效字数 >= 100 |
| **无代理池，IP 被封就跪** | **✅ proxy_manager.py 动态代理池** |
| **详情页失败直接丢弃** | **✅ keep_on_detail_failed 回落** |
| **正文含导航/广告噪音** | **✅ RinseRule 清洗 DSL** |
| **正文提取成功率低** | **✅ 多提取器投票（4个提取器）** |
| **搜索源太少（4个）** | **✅ 扩展到 9 个** |
| **输出目录还在测试文章** | **✅ 改到 文章/** |

## 九、给下个 Agent 的建议

1. **搜索扩展** — 加新源非常容易（`sources/` 下新建文件 + `@register_source`）。候选：搜狗（反爬严格）、掘金、CSDN、36氪、GitHub Trending
2. **Web UI** — 加 Flask/FastAPI 前端，浏览器输入关键词出结果，是最有实用价值的二开方向
3. **AI 摘要** — 爬完内容调 LLM API 自动生成摘要，产出更可直接用
4. **定时任务** — cron 定时跑关键词，自动攒成日报/周报
5. **微信源** — 当前用 Playwright + Sogou 跳转提取，如果搜狗反爬升级可能需要找替代方案
6. **测试方式** — `PYTHONIOENCODING=utf-8 python hotspot_crawler.py <关键词> --max 1 --delay 0.5 --browser never` 快速验证
7. **GitHub** — `https://github.com/2820717824-source/-TAT`，推送前 `git pull --rebase`
