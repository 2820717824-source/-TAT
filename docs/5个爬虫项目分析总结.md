## 逐个分析

### 1. image_crawler — 图片专项爬虫

**架构**：两层结构 — `image_crawler.py`（tkinter GUI）+ `crawler_engine.py`（引擎）

**JS 渲染**：
- 先用 `StaticCrawler`（requests + BeautifulSoup）尝试
- 检测到 SPA（`id="__next"`、`id="__nuxt"` 等）或图片数不足时，自动回退到 `JSCrawler`（Playwright）
- Playwright 配置了完整的隐身上下文：覆盖 `navigator.webdriver`、`navigator.plugins`、`navigator.languages`，注入 `window.chrome`，劫持 `permissions.query`

**亮点**：
- SPA 检测启发式算法（检查框架根节点）
- 10 种图片提取策略（srcset、懒加载、background-image、picture/source、video poster、og:image 等）
- "显示更多"自动点击引擎（19 种匹配策略：文本、CSS 类、aria-label、data-action）
- 翻页检测（CSS 类、link rel="next"、文本关键词）

**缺点**：专为图片设计，不适合通用文章爬取；无速率限制；无代理支持

**可借鉴**：Playwright 隐身配置、SPA 检测、图片提取策略

### 2. python_spider — 爬虫教学示例集

**架构**：零散的教学脚本，无共享框架。覆盖 urllib、requests、Selenium、Ajax 分析、MongoDB 存储

**JS 渲染**：
- 淘宝爬虫（`Taobao_ms.py`）：Selenium Chrome 无头，WebDriverWait 显式等待
- 头条爬虫（`分析Ajax来爬取今日头条街拍美图.py`）：纯 requests 分析 Ajax JSON API，无需浏览器
- Selenium 用法（`Selenium用法.py`）：基础 API 教学

**亮点**：
- Ajax 参数逆向工程技术（分析 `count`、`offset`、`cur_tab` 等参数）
- 网页交互模拟（页码输入 + 点击提交）

**缺点**：硬编码 cookie 会过期；无错误处理；无扩展性

**可借鉴**：Ajax 接口参数逆向分析思路

### 3. scrapy+phantomjs+selenium — Scrapy 集成方案

**架构**：Scrapy 框架 + 自定义 `JavaScriptMiddleware` 下载中间件

**JS 渲染**：PhantomJS（2018 年已停用）。中间件对**每个 URL 刷新页面 50 次**，每次间隔 300-600 秒，拼接 50 次 page_source 来捕获异步内容。每个请求耗时 4-8 小时。

**缺点**：完全不可用。PhantomJS 已死，50 次刷新方案极其低效，路径硬编码，代码混乱。

**可借鉴**：零。唯一有价值的概念是"Scrapy + 浏览器中间件"的思路，但实现不能用。

### 4. universal-crawler — ⭐ 架构最强的通用爬虫框架

**架构**：配置驱动，YAML 文件控制一切。模块化设计：

```
crawler/
  engine.py           核心编排器
  config.py           类型化配置（YAML → dataclass）
  fetcher.py          httpx HTTP 客户端（重试 + 超时 + 编码检测）
  browser_fetcher.py  Playwright 动作框架（滚动/点击/填写/等待）
  parser.py           CSS/XPath/正则/JSONPath 提取器
  paginator.py        4 种分页策略
  storage.py          JSONL/CSV/SQLite 持久化
  dedupe.py           去重（SHA256）
  cleaner.py          文本清洗（HTML 反转义、空白规范化、跟踪参数移除）
  media.py            富媒体下载（图片/音频/视频/字幕/附件）
  attachment_parser.py 附件解析（PDF/docx/xlsx/csv/txt）
  run_state.py        运行状态 + 失败恢复
```

**JS 渲染**：Playwright（Chromium/Firefox/WebKit），YAML 配置驱动：
```yaml
browser:
  enabled: true
  engine: "chromium"
  headless: true
  wait_until: "networkidle"
  actions:
    - type: "scroll"
      times: 5
      pause: 1
```

**亮点**：
- 完全类型化，配置驱动（加新任务不需要写代码）
- 4 种分页策略（page_param / url_template / next_link / cursor）
- 失败恢复（已完成列表 + 失败队列 + 重试失败优先）
- 去重（内存 / 持久化 SHA256）
- 速率限制（delay_min/delay_max + 指数退避重试）
- 附件解析（PDF/Word/Excel 等）
- 日志系统（请求日志 + 错误日志 + 运行摘要）

**缺点**：
- **无 Playwright 隐身上下文**（容易被检测为爬虫）
- 反爬模块的配置字段定义了但未实现（UA 池、代理轮换、指纹）
- 文档只有英文

**可借鉴**：整体架构、YAML 配置系统、模块分离、失败恢复、分页抽象、去重系统

### 5. javaCrawling — 企业级爬虫管理平台

**架构**：Spring Boot 1.4 + WebMagic（Java 版 Scrapy）。两个 Maven 模块：`middletier`（爬虫引擎）+ `boss`（Web 管理后台）

**JS 渲染**：
- `SeleniumDownloader`：Chrome WebDriver，连接池（最多 5 实例），通过 `document.documentElement.outerHTML` 获取
- `WeChatSeleniumDownloader`：混合策略 — 微信公众号用 Selenium，其他回退 HttpClient
- 支持 Firefox/Chrome/PhantomJS/RemoteWebDriver 切换

**亮点**：
- **动态代理池**（`NetPoxyService` 从 data5u API 获取动态代理 IP，请求间轮换）
- Web 管理 UI（添加任务、监控状态）
- 混合下载器策略（按 URL 模式选择浏览器或 HTTP）
- 内容清洗管道（`RinseService` 清理 HTML、修复图片相对路径）
- Redis 调度 + 去重

**缺点**：Spring Boot 1.4（2016），WebMagic 0.7.x 过时；XML 配置冗长；部分代码混乱

**可借鉴**：动态代理池、混合下载器策略、内容清洗管道

## 对 hotspot_crawler.py 升级的建议

### 首选架构蓝本：universal-crawler

它和我们的需求最匹配——通用文章爬取，不是图片专项。模块化设计可以直接复用：

```
hotspot_crawler.py  (保持入口不变)
engine.py           来自 universal-crawler，添加多数据源编排
fetcher.py          来自 universal-crawler（httpx + 重试退避）
browser_fetcher.py  来自 universal-crawler（Playwright 动作框架）
                    合并 image_crawler 的隐身上下文配置
parser.py           来自 universal-crawler（CSS/XPath/正则/JSONPath）
paginator.py        来自 universal-crawler（4 种分页策略）
storage.py          来自 universal-crawler（多后端 + 去重）
proxy.py           借鉴 javaCrawling 的动态代理池模式
```

### 需要从其他项目补充的能力

| 缺失能力 | 来源项目 | 说明 |
|---------|---------|------|
| Playwright 隐身上下文 | image_crawler | 覆盖 webdriver/plugins/languages，防检测 |
| 动态代理轮换 | javaCrawling | 从代理 API 获取 IP，请求间切换 |
| SPA 检测启发式 | image_crawler | 自动判断页面是否需要浏览器渲染 |
| Ajax 接口逆向 | python_spider | 部分站点直接调 API 比渲染 HTML 更高效 |

### 当前爬虫的短板 vs 升级方案

| 当前问题 | 升级方案 |
|---------|---------|
| requests 拿不到 JS 渲染内容 | 添加 Playwright browser_fetcher，YAML 配置开关 |
| --url-file 只能手动列 URL | 添加分页策略 + 规则配置，自动发现文章 |
| 爬崩了得重来 | 添加失败恢复 + 去重，断点续爬 |
| 参数校验简陋 | 统一 YAML 配置 + dataclass 校验 |
| 无代理机制 | 添加动态代理池，IP 被封自动切换 |
| 爬到的内容含导航噪音 | 添加 cleaner + 更精细的提取规则 |
