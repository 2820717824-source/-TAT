> 编写时间：2026-06-23

## 一、当前爬虫的短板清单

| 短板 | 场景 | 严重程度 |
|------|------|:-------:|
| requests 拿不到 JS 渲染的页面 | 大部分新闻站（新浪财经、中国经济时报等） | 🔴 致命 |
| 无代理机制，IP 被封就跪 | 高频请求触发反爬 | 🔴 致命 |
| 无隐身配置，容易被检测为爬虫 | 头条号、知乎等反爬严格的站点 | 🟠 严重 |
| 无失败恢复，爬到一半断了重来 | 长列表爬取 | 🟠 严重 |
| 无去重，重复爬相同内容 | 多次运行同一关键词 | 🟡 中等 |
| 无智能降级，全爬或全不爬 | 混合了静态/动态页面的结果列表 | 🟡 中等 |
| 正文提取不够干净，带导航噪音 | 部分站点正文混杂页脚广告 | 🟡 中等 |
| 搜索源固定三个，新增源要改代码 | 想加微博热搜、今日头条等 | 🟡 中等 |
| 输出只有 Markdown 一种格式 | 需要 JSON/CSV 做数据分析 | 🟢 轻微 |
| 参数全靠命令行传 | 多个参数保存/复用不方便 | 🟢 轻微 |

## 三、分模块优化清单

### 模块 1：搜索器（HotSearcher）—— 多源可扩展

**改造内容：**
- 搜索源改为 **插件式注册**，新增源只需加一个类 + 配置，不改核心代码
- 预置源：百度热搜、知乎热榜、必应搜索、微博热搜（新增）
- 搜索前先搜一下看结果质量，值不值得爬

**参考：** universal-crawler 的配置驱动模式

### 模块 2：获取器 —— 双引擎 + 智能降级（本轮核心）

**新增 `browser_fetcher.py`：**
- **Playwright 渲染引擎**（参考 image_crawler）
  - 隐身配置：覆盖 `navigator.webdriver`、`navigator.plugins`、`navigator.languages`
  - 注入 `window.chrome`，劫持 `permissions.query`
  - `--disable-blink-features=AutomationControlled`
  - 中文语言、桌面 Viewport（1920x1080）
  - 等 `networkidle` 后再拿 HTML
- **动作脚本**（参考 universal-crawler）
  - 滚动懒加载
  - 点击"展开全文""显示更多"
  - 等待特定元素出现

**智能降级策略：**
```
1. requests 直连 → 内容够用？→ 走老路（快路径）
2. 内容太短/检测到 SPA（__next__ 等）→ 走 Playwright（慢路径）
3. Playwright 还拿不到？→ 标记失败，记录原因
4. --browser 参数 → 强制所有页面走 Playwright
```

**参考文件：**
- `image_crawler-main/image_crawler-main/crawler_engine.py`（隐身 + SPA 检测）
- `universal-crawler-main/universal-crawler-main/crawler/browser_fetcher.py`（动作框架）

### 模块 3：反爬对抗

**动态代理池**（参考 javaCrawling 的 NetPoxyService）：
- 从代理 API（data5u/快代理/自建）获取代理 IP 列表
- 请求间轮换 IP
- 失败时自动换下一个

**请求指纹随机化：**
- UA 池已有了，但需要扩充到 20+ 个
- 添加 Sec-CH-UA、Accept 等头部的随机组合
- 请求间隔随机化（不是固定延迟，而是在 1-3 秒间随机）

**Cookie 管理：**
- Session 持久化，跨请求保持 cookie
- 部分站点需要首次访问拿 cookie 再做第二次请求

### 模块 4：提取器（ArticleCrawler）—— 多策略 + 更干净的正文

**改进 Readability 提取：**
- 当前已用 readability-lxml，但有的站点提取效果不好
- 添加备选提取器：
  - **Readability**（主方案）
  - **BeautifulSoup 文本密度**（已实现）
  - **Trafilatura**（备选，可装可不装）
  - **justext**（备选）
- 取多个提取器的结果中最长的那个

**内容清洗（参考 universal-crawler 的 cleaner.py）：**
- 去掉导航栏、版权声明、广告等噪音
- 修复相对路径的图片链接
- 统一编码（已有部分，需加强）

### 模块 5：存储 —— 多格式支持

**当前：** 只有 Markdown + _summary.md
**升级：**
- Markdown（默认）
- JSONL（每行一个 JSON，适合数据分析）
- CSV（表格形式）
- SQLite（可查询、去重更方便）

**去重（参考 universal-crawler 的 dedupe.py）：**
- SHA256 内容哈希去重
- 跨运行持久化（SQLite 或文件）
- 相同 URL 已爬过 → 跳过

**失败恢复（参考 universal-crawler 的 run_state.py）：**
- 记录已完成列表
- 记录失败队列
- 断点续爬：重启时跳过已成功的，重试失败的

### 模块 6：配置 —— YAML 驱动

**新增 `config.yaml`：**
```yaml
keyword: "冻零树"
sources:
  - baidu_hot
  - zhihu_hot
  - bing_news
  - weibo_hot
crawler:
  max_per_source: 15
  delay_min: 1.0
  delay_max: 3.0
  browser: auto  # auto / always / never
  proxy: "http://proxy_pool:8080"
output:
  formats: [md, jsonl]
  dir: "C:/Users/24394/Desktop/爬虫热点/文章"
  date_subdir: true
```

CLI 参数与 YAML 配置合并，CLI 参数优先：
```bash
# 使用配置文件
python hotspot_crawler.py --config config.yaml

# 使用 CLI 参数（覆盖 config 中的 keyword）
python hotspot_crawler.py 冻零树 --config config.yaml --max 10
```

### 模块 7：日志 —— 可追踪

**当前：** rich console 打印，终端的 GBK 还会崩
**升级：**
- 文件日志（logging 模块，UTF-8）
- 请求日志：每个 URL + 状态码 + 耗时
- 错误日志：保留完整 traceback
- 运行摘要：成功/失败/总计/平均耗时

```
browser_fetcher.py       ← 新增
```

### Phase 2：反爬对抗

```

user_agent.py            ← 新增（UA 池扩充 + 指纹随机化）
hotspot_crawler.py       ← 改：集成 proxy
```

### Phase 3：工程化

```
config.yaml              ← 新增
dedupe.py                ← 新增
run_state.py             ← 新增
storage.py               ← 新增（多格式支持）
hotspot_crawler.py       ← 改：配置驱动 + 恢复 + 去重
```

### Phase 4：内容质提升

```
cleaner.py               ← 新增（内容清洗）
extractor.py             ← 新增（多提取器 + 投票）
```

### Phase 5：搜索源扩展（✅ 已完成 → 批次 3）

```
sources/                 ← 已创建目录
sources/__init__.py      ← 注册机制 + importlib 自动发现
sources/baidu.py         ← 从 HotSearcher 拆分
sources/zhihu.py         ← 同上
sources/bing.py          ← 同上
sources/weibo.py         ← 新增（微博热搜，通过 weibo.com/ajax/side/hotSearch）
searcher.py              ← 编排器（替代 HotSearcher，策略链模式）
hotspot_crawler.py       ← 改：移除 HotSearcher，使用新 Searcher
```

**注意：实施过程将原 5 个 Phase 重组为 3 个批次，搜索源扩展为第 3 批。**
**新增源流程：sources/ 下建文件 → 类加 @register_source → 自动注册（无需改 __init__.py）**
